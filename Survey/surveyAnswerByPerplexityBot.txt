# Family Protection Survey — Analysis & Answers for 3 Personas

Here is a complete breakdown of all **7 questions** across 3 pages, answered for each persona with reasoning.

***

## The 3 Personas

| # | Persona | Profile |
|---|---------|---------|
| 1 | **Salaried Employee** | Mid-level IT/govt/private sector worker, ₹5–15L/year, has EPF, bank account, maybe 1 LIC policy, wife & kids |
| 2 | **Businessman** | SME owner, multiple bank accounts, property, business assets, more financially complex |

### Q1. If something unexpected happens to you (sudden death), would your family know how to access all your assets?

| Option | Salaried Employee | Businessman | NRI |
|--------|:-:|:-:|:-:|
| Yes, they know everything | | | |
| **They know some details** | ✅ | | |
| They know very little | | | ✅ |
| No, they don't know anything | | | |
| **I am not sure** | | ✅ | |

**Salaried Employee → "They know some details"**
> A typical salaried Indian has told his wife about the main bank account and maybe LIC policy, but EPF UAN, NPS PRAN, FD numbers, demat account — these are often untouched. He knows *some* is shared but not all. "They know everything" is overconfident; "very little" is too pessimistic for someone organized enough to have nominees filed.

**Businessman → "I am not sure"**
> A businessman has a web of accounts — current accounts, GST registration, property deeds, partnership deeds, business insurance. He likely hasn't sat down to document it all. Even his spouse may know the basics but not the full picture. "Not sure" is the most honest and realistic answer here.

**NRI → "They know very little"**
> An NRI has the most fragmented financial life — NRE/NRO accounts in India, foreign bank accounts, 401k/pension abroad, property back home managed by relatives, mutual funds, PPF. His India-based family doesn't know the overseas details, and his overseas family doesn't know the India details. "Very little" is the realistic answer.

### Q2. After Death: Do you want to share your financial details with loved ones in case of death?

| Option | Salaried Employee | Businessman | NRI |
|--------|:-:|:-:|:-:|
| **Yes, but ONLY after my death.** | ✅ | ✅ | ✅ |
| Yes, I want to share now. | | | |
| No, I have already shared. | | | |
| No, I don't want to share. | | | |

**All three → "Yes, but ONLY after my death."**
> This is the near-universal Indian sentiment. Privacy of financial details is a strong cultural norm in India — sharing bank balances or loan details with even close family while alive is considered uncomfortable or invites interference. But everyone wants their family protected *after* they're gone. "Share now" feels invasive; "already shared" is rare; "don't want to share" would leave family helpless. This option perfectly matches Indian sensibility across all three personas.

### Q3. Do you think it would be helpful to keep all your asset details organized in one place?

| Option | Salaried Employee | Businessman | NRI |
|--------|:-:|:-:|:-:|
| Yes both in Paper & Digital format | | | |
| **Yes, Digital only** | ✅ | | |
| Yes, Physical paper only | | | |
| **Yes, Digitally only if it remains completely private while I'm alive** | | | ✅ |
| No, I don't trust digital platform | | | |
| No, I don't want to store anywhere | | | |
| **Yes both in Paper & Digital format** | | ✅ | |

**Salaried Employee → "Yes, Digital only"**
> A young-to-middle-aged salaried employee is comfortable with apps (Groww, CAMS, DigiLocker already exist). He's already doing digital KYC and e-filing ITR. Paper feels old-fashioned. He'd prefer a clean digital record. He's not paranoid about digital security if the app is known/trusted.

**Businessman → "Yes both in Paper & Digital format"**
> A businessman deals with legal documents, CA filings, property papers — the physical paper world is very real to him. He'd want a digital backup AND physical copies because property deeds, partnership agreements, etc. have legal validity on paper. He doesn't fully trust "digital only" for high-value assets.

**NRI → "Yes, Digitally only if it remains completely private while I'm alive"**
> An NRI is tech-savvy and fine with digital, but has a heightened concern about privacy — his financial life spans two countries and any data leak is more damaging. He specifically wants the assurance of privacy during his lifetime. This option is the most fitting.

### Q4. How much details would you like to store with a trusted website/app company?

| Option | Salaried Employee | Businessman | NRI |
|--------|:-:|:-:|:-:|
| Yes, comfortable with all details (Login + Password + Documents) | | | |
| **Yes, but ONLY basic financial details (account numbers) — NO PASSWORD, NO DOCUMENTS** | ✅ | | |
| **Maybe, if the platform is highly secure and encrypted.** | | ✅ | ✅ |
| No, I don't want to store any financial information online | | | |

**Salaried Employee → "Only basic financial details — NO PASSWORD, NO DOCUMENTS"**
> A salaried Indian has been warned by banks and WhatsApp forwards never to share passwords. He's cautious but practical — account numbers and policy numbers are fine to store. Uploading full documents or passwords feels like too much risk. This middle-ground option is very "average Indian" behavior.

**Businessman → "Maybe, if the platform is highly secure and encrypted"**
> A businessman is risk-aware. He'd want to evaluate the platform thoroughly before committing. His data is more sensitive (business accounts, property docs), so he won't commit to a hard yes, but won't say no either. "Maybe with security assurance" is his answer.

**NRI → "Maybe, if the platform is highly secure and encrypted"**
> NRIs are generally more aware of data breaches (GDPR, US data laws, etc.). They've used international apps and know what "encrypted" means. They're open to it but need platform credibility. Same answer as businessman but for different reasons — tech awareness vs. risk management.

### Q5. In case of your sudden death, do you want your nominated family member to receive a confidential email or postal letter (Speed Post) containing all financial details?

| Option | Salaried Employee | Businessman | NRI |
|--------|:-:|:-:|:-:|
| Yes, confidential (Speed Post) postal letter is preferred only | | | |
| **Yes, confidential E-Mail is preferred only.** | ✅ | | |
| **Yes, I want both Speed Post & E-Mail.** | | ✅ | |
| No, I don't want any release | | | |
| **Yes, confidential E-Mail is preferred only.** | | | ✅ |

**Salaried Employee → "Yes, confidential E-Mail is preferred only"**
> A salaried employee's family is likely to have a smartphone and email access. Email is immediate, free, and feels secure enough. Speed Post feels old and unreliable — what if the family has moved? Email is the default communication channel for this demographic.

**Businessman → "Yes, I want both Speed Post & E-Mail"**
> A businessman deals with legal and formal communications regularly. He knows email can be hacked or missed. For something this critical, he'd want both channels — the email for speed and the postal letter as a legally credible physical document (useful for bank/property claims). Redundancy = security in his mindset.

**NRI → "Yes, confidential E-Mail is preferred only"**
> For an NRI, Speed Post to an international address is unreliable, expensive, and slow. Email is the obvious choice. His family in India or abroad is definitely email-enabled. Speed Post makes no practical sense for someone living outside India.

### Q6. If your financial data is shared securely with loved ones via confidential Speed Post, will you be willing to pay a one-time fee?

| Option | Salaried Employee | Businessman | NRI |
|--------|:-:|:-:|:-:|
| Yes, as long as it's secure and reliable | | | |
| **Yes, but it depends on the price.** | ✅ | | ✅ |
| **Maybe, I'd like more details first.** | | ✅ | |
| No, I will only use if it's FREE | | | |
| No, I don't want this service | | | |

**Salaried Employee → "Yes, but it depends on the price"**
> A salaried employee is value-conscious. He's willing to pay for genuinely useful services (he pays for Hotfix, Zomato Gold, etc.) but price sensitivity is real. He won't say no outright, but ₹499 vs ₹4999 one-time would make a big difference. "Depends on price" is the most honest middle-class Indian answer.

**Businessman → "Maybe, I'd like more details first"**
> A businessman doesn't make financial decisions without due diligence. He'd want to know: Who runs this company? What's the legal guarantee? How is the data stored? What's the refund policy? He's not price-sensitive, but trust and clarity come before money.

**NRI → "Yes, but it depends on the price"**
> An NRI earns in foreign currency, so a one-time Indian rupee fee is small. But they're also skeptical of Indian startups' reliability from abroad. They're willing to pay, but want the price to be justified. "Depends on price" reflects cautious optimism.

### Q7. What is your biggest concern about such a service? (Multiple choice)

| Concern | Salaried Employee | Businessman | NRI |
|---------|:-:|:-:|:-:|
| I don't find it useful enough | | | |
| **Data Security / Privacy / Misuse of information** | ✅ | | ✅ |
| Price of the Service | | | |
| Wrong release to nominee | | | |
| **Reputation & Trust in the Company/Service** | | | |
| **Reliability of Company & Service** | | ✅ | |
| I have no concerns | | | |

**Salaried Employee → "Data Security / Privacy / Misuse of information"**
> The average salaried Indian has been through OTP frauds, SIM swap scams, KYC frauds. His number one fear is: "What if this company leaks my data or misuses it while I'm alive?" This is the most common and relatable concern for this demographic.

**Businessman → "Reliability of Company & Service"**
> A businessman's concern is operational: "Will this company still exist 20 years from now when my family actually needs this?" Startups shut down, pivot, or get acquired. For a long-term, post-death service, reliability and longevity of the service provider is the #1 worry for a practical businessman.

**NRI → "Data Security / Privacy / Misuse of information"**
> An NRI, having lived in GDPR/privacy-law aware countries, is acutely sensitive to data privacy. With financial data spanning two countries, a breach is doubly damaging. Data security is the top concern, same as the salaried employee but for more informed reasons.
